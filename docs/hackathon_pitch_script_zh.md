# 黑客松汇报基础讲稿

大家好，我们做的是一个面向世界杯比赛视频的自动解说生成系统。

这不是一个简单的“把图片丢给大模型描述”的 demo。足球比赛有几个非常现实的问题：一场比赛帧数非常多，关键事件很稀疏；进球后通常还有庆祝和回放，如果切分不好，模型会把同一个进球讲好几遍；同时，普通大模型很容易把解说写成平淡的画面说明，缺少电视解说的节奏和现场感。

我们的做法是把整条链路拆开。

第一步，用户提供已经切好的视频帧和 manifest。第二步，系统用低帧率滑窗粗扫全场，给每一帧打上事件标签。第三步，把粗扫出来的事件按时间和类型聚合，尤其是把进球、庆祝、回放整理成一个更适合解说的事件包。第四步，可选地调用视觉模型对 goal 候选做二次校验。最后，系统用事件信息、抽样视频帧、风格配置和比赛上下文生成中英文解说稿。

这个项目最重要的设计是解耦。

风格不是写死在代码里，而是 `StyleProfile`；比赛事实不是散落在 prompt 里，而是 `MatchContext`；视频帧输入是 manifest；事件输出是结构化 JSON；每次 API 调用都有 trace。这样一来，后续要换模型、换赛事、换风格、换切帧方式，都不需要推倒重来。

这次我们新增了比赛关键信息注入。比如当前视频明确告诉模型：比赛是德国队对库拉索队，蓝黄球衣是库拉索，不是哥伦比亚，也不是巴拉圭。这个信息会进入英文生成和中文翻译，避免模型在长链路里把队名写错。

我们也新增了两种风格。第一种是故事化轻幽默风，适合黑客松演示和集锦；第二种是高阶转播回放风，适合进球、扑救、VAR 这种需要先调用直播瞬间，再利用回放解释技术细节的场景。

工程上，系统支持断点续跑、并发调用、进度条、缓存和中间结果审计。即使一场比赛要调用几百次 API，也能知道当前跑到哪里、哪个事件出了问题、某一次输出来自哪些帧和哪些 prompt。

最终我们希望它成为一个视频解说生产 Harness：输入比赛帧，选择赛事上下文和解说风格，输出可审核、可修改、可配音的中英文解说脚本。

如果继续做下去，我们会优先补三件事：第一，进球去重和回放归并做得更强；第二，做一个轻量 UI 给用户调风格和事件范围；第三，把结果接到字幕和 TTS，形成完整的视频二创工作流。
