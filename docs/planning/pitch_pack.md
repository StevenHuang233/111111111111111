# Pitch Pack

Use this as the default 10-15 minute presentation structure.

## 30 Second Version

- Problem: what pain or opportunity exists.
- Solution: what the system does.
- Evidence: what the audience can see in the demo or metrics.
- Value: why it matters.

## 10-15 Minute Structure

| Segment | Time | Purpose | Required Evidence |
| --- | --- | --- | --- |
| Opening | 0:00-1:00 | State problem and value | One concrete user or task scenario |
| Methodology | 1:00-2:30 | Explain workflow/harness thinking | Architecture diagram or loop |
| System | 2:30-5:00 | Show modules and data flow | Trace, tool list, state flow |
| Demo | 5:00-9:00 | Prove it works | Live run or recorded fallback |
| Evaluation | 9:00-11:00 | Defend quality and cost | Metrics, baseline, failure analysis |
| Social Value | 11:00-12:30 | Explain usefulness beyond demo | Target users and impact |
| Team Work | 12:30-13:30 | Prove execution ability | Contribution evidence |
| Limits And Future | 13:30-14:30 | Show rigor | Known limits and next steps |
| Close | 14:30-15:00 | Repeat core claim | One memorable sentence |

## Q&A Prep

- What is the core technical novelty?
- Why is this more than a prompt wrapper?
- What happens when the model is wrong?
- How do you evaluate quality?
- How do you control cost?
- What part is reusable after the event?
- What did each member contribute?
