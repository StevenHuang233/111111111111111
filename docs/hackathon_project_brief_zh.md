# 项目基础描述：世界杯视频解说 Harness

## 一句话介绍

这是一个面向足球比赛视频的多模态解说生成 Harness：输入切好的视频帧，系统先粗扫定位事件，再把进球、回放、庆祝和关键攻防整理成可讲述的事件单元，最后按指定风格生成中英文解说稿。

## 解决的问题

传统大模型问答很难直接处理一整场比赛视频，因为它需要同时解决：

- 视频帧太多，不能一次性全量送入模型。
- 事件边界模糊，进球、回放、庆祝容易被拆开或重复。
- 解说风格难控制，普通输出容易变成平铺直叙的画面描述。
- 生成链路长，需要断点续跑、日志记录和可审计中间结果。

本项目把这些问题拆成可替换模块，便于调参和迭代。

## 当前架构

```text
frame manifest
  -> low-fps coarse scan
  -> event aggregation
  -> commentary unit packing
  -> optional goal verification
  -> visual bilingual generation
  -> reviewable JSON / trace / progress logs
```

## 关键设计

### 1. 高解耦

每层只通过结构化数据交互：

- `frames_manifest.json` 负责视频帧输入。
- `EventCandidate` 负责事件结果。
- `StyleProfile` 负责风格。
- `MatchContext` 负责比赛事实注入。
- `TraceRecorder` 负责调用记录。

### 2. 风格可插拔

风格通过 JSON 配置注入 prompt 和超参数。当前支持：

- 专业电视转播风
- 短视频激情风
- 战术分析风
- 故事化轻幽默风
- 高阶转播回放风

### 3. 比赛信息注入

新增 `MatchContext` 解决队名和语境错误。例如本视频明确注入：

- 德国队 vs 库拉索队
- 蓝黄球衣是库拉索，不是哥伦比亚
- 中文应使用“德国队 / 库拉索队”

这能显著减少模型在翻译或生成时把队名写错的问题。

### 4. 可审计

系统会记录：

- 每一步进度
- 每次 API 调用输入输出
- 中间事件 JSON
- 进球校验报告
- 最终中英文解说稿

这让黑盒生成变成可检查、可复跑、可定位问题的工程流程。

## 演示亮点

- 展示同一个进球在不同风格下的解说差异。
- 展示进球直播 + 回放阶段的双重解说。
- 展示错误队名通过 MatchContext 被纠正。
- 展示断点续跑和进度条日志，说明系统适合长视频处理。
